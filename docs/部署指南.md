# Anchor v0.2 部署指南

## 架构概览

```
┌─────────────────────────────────────────────┐
│                宿主机 / 服务器                 │
│  ┌─────────────┐      ┌───────────────────┐ │
│  │ 桌面/Web 端  │      │  Server Docker    │ │
│  │ 管理界面     │◄────►│  :17421           │ │
│  └─────────────┘      │  --no-local-worker│ │
│                       └─────────┬─────────┘ │
│                                 │ HTTP      │
│                       ┌─────────▼─────────┐ │
│                       │  Worker Docker    │ │
│                       │  注册/心跳/轮询    │ │
│                       └───────────────────┘ │
└─────────────────────────────────────────────┘
```

## Docker 一键部署

### 构建镜像

```bash
docker build -f Dockerfile.server -t anchor-server .
docker build -f Dockerfile.worker -t anchor-worker .
```

### 部署 Server（纯管理面）

```bash
docker run -d --name anchor-server \
  -p 17421:17421 \
  -v anchor-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  anchor-server \
  ./anchor --no-local-worker
```

### 部署 Worker

**macOS Docker Desktop：**
```bash
docker run -d --name anchor-worker \
  -v anchor-worker-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  anchor-worker \
  --core-url=http://host.docker.internal:17421
```

**Linux（使用宿主机 LAN IP）：**
```bash
docker run -d --name anchor-worker \
  -v anchor-worker-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  anchor-worker \
  --core-url=http://192.168.0.107:17421
```

### 使用 docker-compose

```bash
# 仅启动 Server
docker compose up -d server

# 启动 Server + Worker
docker compose --profile worker up -d
```

## 客户端连接

### Web 浏览器
直接访问：`http://192.168.0.107:17421`

### 桌面端（Tauri）
1. 打开应用 → Settings
2. Server 地址输入：`http://192.168.0.107:17421`
3. 点击「保存并刷新」
4. 应用自动重连到远程 Server

### 本地模式（单机使用）
直接运行 `./anchor`，Server 会自动启动内置 Local Worker。

## E2E 验证脚本

```bash
# 1. Server API
curl -s http://192.168.0.107:17421/workers | jq .

# 2. Worker 注册（应看到 remote worker，status=online）
curl -s http://192.168.0.107:17421/workers | jq '.[] | select(.mode=="remote")'

# 3. 创建项目
PROJECT=$(curl -s -X POST http://192.168.0.107:17421/projects \
  -H "Content-Type: application/json" \
  -d '{"name":"test"}')
echo $PROJECT | jq '.id'

# 4. 创建 Run
PROJECT_ID=$(echo $PROJECT | jq -r '.id')
curl -s -X POST "http://192.168.0.107:17421/projects/$PROJECT_ID/runs" \
  -H "Content-Type: application/json" \
  -d '{"tool_template_id":"external-quick","name":"test-run"}' | jq '.id'
```

## 已知限制

| 限制 | 说明 |
|------|------|
| macOS Docker 网络 | Worker 容器通过 `host.docker.internal` 访问宿主机，不能用 `192.168.x.x` |
| 任务分配 | Run → ScanTask → Worker 的完整调度链路尚未实现 |
| Rod 截图 | Docker 中运行 Chromium 需 `--privileged` 或 `--cap-add=SYS_ADMIN` |
| Nuclei 模板 | Worker 镜像未预装 nuclei-templates，需挂载卷或手动安装 |
