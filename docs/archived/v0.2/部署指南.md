# Anchor v0.2 部署指南

## 架构概览

```
┌─────────────────────────────────────────────┐
│                宿主机 / 服务器                 │
│  ┌─────────────┐      ┌───────────────────┐ │
│  │ 桌面/Web 端  │      │  Server Docker    │ │
│  │ 管理界面     │◄────►│  :17421           │ │
│  └─────────────┘      │  --no-local-worker│ │
│                       └─────────┬─────────┘ │
│                                 │ HTTP      │
│                       ┌─────────▼─────────┐ │
│                       │  Worker Docker    │ │
│                       │  注册/心跳/轮询    │ │
│                       └───────────────────┘ │
└─────────────────────────────────────────────┘
```

**核心原则**：Worker 只需能 outbound 访问 Server，Worker 自身**不需要公网 IP**。

---

## 快速开始（Docker Compose）

### 一键启动 Server + Worker + Rangefield

```bash
# 构建镜像 + 启动全部服务
make up-all

# 查看状态
make status

# 查看日志
make logs
make logs-server
make logs-worker

# 停止全部服务
make down-all
```

### 仅启动 Server（无 Worker）

```bash
make up
```

### 启动 Server + Worker（不含 Rangefield）

```bash
docker compose --profile worker up -d
```

## 环境变量参考表

| 变量名 | 用途 | 默认值 | 适用场景 |
|--------|------|--------|----------|
| `ANCHOR_DATA_DIR` | 数据目录 | `~/.anchor` | Server / Worker |
| `ANCHOR_PORT` | Server 监听端口 | `17421` | Server |
| `ANCHOR_CORE_URL` | Worker 连接 Server 的地址 | `http://localhost:17421` | Worker |
| `ANCHOR_WORKER_HOST` | Worker 对外暴露的 hostname | `127.0.0.1` | Worker（远程模式） |

---

## 网络要求

| 方向 | 要求 | 说明 |
|------|------|------|
| Worker → Server | **必须可达** | Worker 主动发起 HTTP 连接到 Server |
| Server → Worker | **不需要** | 所有控制通过 Worker 长轮询实现 |
| 客户端 → Server | **必须可达** | 桌面端或浏览器访问管理界面 |
| Worker 公网 IP | **不需要** | NAT 后的内网机器、家庭 WiFi 笔记本均可 |

> **为什么 Worker 不需要公网 IP？**
> Worker 通过 outbound HTTP 注册到 Server，然后使用长轮询（long-polling）持续拉取任务。Server 从不需要主动连接 Worker，因此 Worker 可以位于任何能访问 Server 的网络环境中。

## Server 单独部署

### Docker

```bash
docker run -d --name anchor-server \
  -p 17421:17421 \
  -v anchor-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  -e ANCHOR_PORT=17421 \
  anchor-server \
  ./anchor --no-local-worker
```

### 二进制直接运行

```bash
# 构建
make build

# 运行（纯管理面，不启动本地 Worker）
./bin/anchor --no-local-worker

# 运行（带本地 Worker）
./bin/anchor
```

## Worker 单独部署

### 场景一：Docker 内网（同 Compose 网络）

适用于 Server 和 Worker 在同一台机器的 Docker 中运行：

```bash
docker run -d --name anchor-worker \
  --network anchor-net \
  -v anchor-worker-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  -e ANCHOR_CORE_URL=http://server:17421 \
  -e ANCHOR_WORKER_HOST=worker \
  anchor-worker \
  ./anchor --worker --core-url http://server:17421
```

> 在同一 Docker Compose 网络中，使用 service 名 `server` 作为 hostname。

### 场景二：公网 VPS（Worker 在云端）

适用于 Server 部署在公网 VPS，Worker 部署在另一台云主机：

```bash
# Worker 侧
./anchor --worker --core-url http://203.x.x.x:17421

# 或使用 Docker
docker run -d --name anchor-worker \
  -v anchor-worker-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  -e ANCHOR_CORE_URL=http://203.x.x.x:17421 \
  anchor-worker \
  ./anchor --worker --core-url http://203.x.x.x:17421
```

> 替换 `203.x.x.x` 为 Server 的实际公网 IP。

### 场景三：家庭 WiFi 笔记本

适用于 Server 在公网 VPS，Worker 在家庭网络内的笔记本：

```bash
# 笔记本上直接运行（无需公网 IP，无需端口映射）
./anchor --worker --core-url http://203.x.x.x:17421
```

> 笔记本只要能访问互联网即可，不需要路由器端口映射，不需要公网 IP。

### 场景四：macOS Docker Desktop

适用于开发调试，Server 和 Worker 都在 macOS Docker Desktop 中：

```bash
docker run -d --name anchor-worker \
  -v anchor-worker-data:/data \
  -e ANCHOR_DATA_DIR=/data \
  -e ANCHOR_CORE_URL=http://host.docker.internal:17421 \
  anchor-worker \
  ./anchor --worker --core-url http://host.docker.internal:17421
```

> `host.docker.internal` 是 Docker Desktop 专有的 DNS 名，解析到宿主机 IP。

## Rangefield 靶场

用于本地测试扫描链路：

```bash
# 启动靶场
make range-up

# 查看靶场状态
make range-status

# 查看靶场日志
make range-logs

# 停止靶场
make range-down
```

靶场包含多个漏洞服务（DVWA、Juice Shop 等），部署在 Docker 网络 `anchor-net` 中，Worker 可直接访问。

## 构建镜像

```bash
# Server 镜像
docker build -f Dockerfile.server -t anchor-server .

# Worker 镜像
docker build -f Dockerfile.worker -t anchor-worker .
```

## 客户端连接

### Web 浏览器
直接访问：`http://server-ip:17421`

### 桌面端（Tauri）
1. 打开应用 → Settings
2. Server 地址输入：`http://server-ip:17421`
3. 点击「保存并刷新」
4. 应用自动重连到远程 Server

### 本地模式（单机使用）
直接运行 `./anchor`，Server 会自动启动内置 Local Worker。

## E2E 验证

```bash
# 1. 检查 Server API
curl -s http://localhost:17421/projects | jq .

# 2. 检查 Worker 注册（应看到 remote worker，status=online）
curl -s http://localhost:17421/workers | jq '.[] | select(.mode=="remote")'

# 3. 创建测试项目
PROJECT=$(curl -s -X POST http://localhost:17421/projects \
  -H "Content-Type: application/json" \
  -d '{"name":"test","organization":"test","purpose":"E2E验证"}')
PROJECT_ID=$(echo $PROJECT | jq -r '.id')
echo "Project ID: $PROJECT_ID"

# 4. 添加目标
curl -s -X POST "http://localhost:17421/projects/$PROJECT_ID/targets" \
  -H "Content-Type: application/json" \
  -d '{"type":"domain","value":"example.com"}'

# 5. 触发资产发现
curl -s -X POST "http://localhost:17421/projects/$PROJECT_ID/workflows/asset-discovery"
```

## 已知限制

| 限制 | 说明 |
|------|------|
| macOS Docker 网络 | Worker 容器通过 `host.docker.internal` 访问宿主机，不能用 `192.168.x.x` |
| Rod 截图 | Docker 中运行 Chromium 需 `--privileged` 或 `--cap-add=SYS_ADMIN` |
| Nuclei 模板 | Worker 镜像已预装 nuclei-templates（构建时 `nuclei -update-templates`） |
| 跨主机文件 | Worker 和 Server 共享数据时，跨主机部署应通过 HTTP 传输文件内容，而非共享 volume |
