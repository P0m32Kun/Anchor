# v0.2 执行计划

> 按里程碑 M0→M5 顺序推进，每个阶段包含目标、验收标准、端到端测试。出现 bug 时自动修复。每阶段完成后更新状态标记。
> 当前状态：M0 未开始

---

## 总体原则

1. **先数据层，后逻辑层，最后前端**：每个里程碑先改数据库/model，再写后端逻辑，最后做前端页面
2. **端到端测试驱动**：每个功能必须有至少一个从 API 到数据库的 E2E 测试
3. **Bug 即停即修**：测试失败时暂停当前任务，定位 root cause，修复后再继续
4. **最小可用**：不追求完美实现，先跑通主流程，边缘 case 后置
5. **状态标记**：每阶段完成后更新本文档顶部的「当前状态」

---

## 通用工具与约定

### 端到端测试方式

每个里程碑的 E2E 测试使用 `curl` + `jq` + `sqlite3` CLI 组合，验证：
1. API 返回正确
2. 数据库状态正确
3. 文件系统状态正确（截图、归档等）

### Bug 修复流程

```
测试失败 → 查看完整错误输出 → 定位文件 → 分析 root cause → 修复 → 重新测试 → 通过
```

使用 `rtk run -c "..."` 保留完整错误输出用于调试（不触发 rtk 的类型化过滤）。

### 状态标记格式

```markdown

**状态**: ✅ 已完成 / 🔄 进行中 / ⏳ 未开始
**开始时间**: YYYY-MM-DD
**完成时间**: YYYY-MM-DD
**阻塞问题**: 无 / xxx
```

## M1: 独立 Worker

**状态**: ✅ 已完成
**开始时间**: 2026-04-27
**完成时间**: 2026-04-27
**阻塞问题**: 无

### 目标

将 v0.1 的同进程 Worker 升级为独立进程（本地模式）。核心服务器能 spawn 本地 Worker 子进程，Worker 能执行工具并回传结果。

### 具体任务

#### 后端（Go）

- [ ] 数据库迁移：新增 `worker_nodes` 表
- [ ] 数据库迁移：新增 `worker_health_checks` 表
- [ ] 新增 Model：`WorkerNode`、`WorkerHealthCheck`
- [ ] Worker 独立进程：`cmd/worker/main.go` 或 `./anchor --worker` 模式
- [ ] 核心服务器 spawn 本地 Worker：`runner.SpawnLocalWorker()`
- [ ] Worker stdout 解析：获取 endpoint（`WORKER_READY http://localhost:{port}`）
- [ ] Worker 进程监控：崩溃检测 + 自动重启（3 次 backoff）
- [ ] Worker 健康检查：工具版本检测、Rod 状态检测
- [ ] 新增 API：`GET /workers`、`GET /workers/:id`、`POST /workers/:id/stop`
- [ ] 任务执行迁移：从同进程改为通过 HTTP 下发给本地 Worker

#### 前端（React）

- [ ] Workers 页面：显示本地 Worker 状态（运行中/已停止/错误）
- [ ] Workers 页面：显示工具健康状态列表
- [ ] Workers 页面：启动/停止按钮
- [ ] Dashboard 顶部状态条：显示在线 Worker 数量

### 验收标准

1. 桌面端启动后，本地 Worker 自动启动
2. Workers 页面显示本地 Worker 为"运行中"
3. 健康检查显示 Subfinder、httpx、Naabu、Nuclei、Rod 状态
4. 执行一次资产发现扫描，任务由本地 Worker 执行
5. Worker 进程异常退出后，系统自动重启（最多 3 次）
6. 用户点击"停止"后，Worker 优雅退出（SIGTERM → SIGKILL）

### 端到端测试

```bash
# Test 1: Worker 自动启动
sleep 5
curl -s http://localhost:17421/workers | jq '.[0].status'
# 期望输出: "running"

# Test 2: 健康检查
curl -s http://localhost:17421/workers/health | jq '.tools[].name'
# 期望输出包含: subfinder, httpx, naabu, nuclei, rod

# Test 3: 任务执行（资产发现）
# 先创建一个项目和一个目标域名
curl -s -X POST http://localhost:17421/projects \
  -H "Content-Type: application/json" \
  -d '{"name":"test-m1","organization":"test"}' | jq -r '.id'
# 用返回的 project_id 创建目标
curl -s -X POST http://localhost:17421/projects/{id}/targets \
  -H "Content-Type: application/json" \
  -d '{"type":"domain","value":"example.com"}'
# 启动资产发现
curl -s -X POST http://localhost:17421/projects/{id}/workflows/asset-discovery
# 等待任务完成
sleep 30
# 查询任务状态
curl -s http://localhost:17421/scan-tasks/{task_id} | jq '.status'
# 期望输出: "completed" 或 "partial_success"

# Test 4: Worker 进程存在
ps aux | grep "anchor --worker" | grep -v grep
# 期望返回 0（有进程）

# Test 5: Worker 重启
kill -9 $(pgrep -f "anchor --worker")
sleep 3
ps aux | grep "anchor --worker" | grep -v grep
# 期望返回 0（进程已重启）
```

## M2: Worker 最小闭环（远程 Worker）

**状态**: 🔄 进行中
**开始时间**: 2026-04-27
**完成时间**: —
**阻塞问题**: 无

### 目标

支持远程 Worker 注册、心跳、任务拉取、结果回传。外网场景可用。

### 具体任务

#### 后端（Go）

- [ ] 远程 Worker token 生成：`POST /workers/register`（生成 token）
- [ ] 远程 Worker 启动：`./anchor --worker --core-url http://... --token xxx`
- [ ] Worker 长轮询拉取任务：`GET /workers/:id/tasks/poll`
- [ ] Worker 心跳上报：`POST /workers/:id/heartbeat`
- [ ] Worker 结果回传：`POST /tasks/:id/result`
- [ ] 核心服务器从 Worker 拉取文件：`GET /workers/:id/files/:path`
- [ ] Worker 端 Scope Check 加固
- [ ] token 撤销：`POST /workers/:id/revoke`
- [ ] NetworkProfile 外网/内网/受限/靶场标记

#### 前端（React）

- [ ] Workers 页面：远程 Worker 注册入口（生成 token、复制命令）
- [ ] Workers 页面：显示远程 Worker 状态、心跳时间
- [ ] Dashboard 状态条：NetworkProfile 标签（外网/内网/受限/靶场）

### 验收标准

1. 可以生成远程 Worker token 并复制启动命令
2. 在另一台机器上启动远程 Worker，成功注册到核心服务器
3. 远程 Worker 能长轮询拉取任务并执行
4. 任务结果回传到核心服务器，前端能看到
5. token 撤销后，远程 Worker 不再能拉取任务（返回 401）
6. 本地 Worker 和远程 Worker 可以共存

### 端到端测试

```bash
# Test 1: 生成远程 Worker token
curl -s -X POST http://localhost:17421/workers/register \
  -H "Content-Type: application/json" \
  -d '{"name":"remote-test"}' | jq -r '.token'
# 保存 token

# Test 2: 模拟远程 Worker 注册（用 curl 模拟长轮询）
TOKEN="xxx"
# 先心跳
curl -s -X POST http://localhost:17421/workers/{worker_id}/heartbeat \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"status":"idle","capabilities":["subfinder","httpx"]}'
# 期望返回 200

# Test 3: 长轮询（无任务时返回 204）
curl -s -w "%{http_code}" -o /dev/null \
  "http://localhost:17421/workers/{worker_id}/tasks/poll?timeout=5s" \
  -H "Authorization: Bearer $TOKEN"
# 期望输出: 204

# Test 4: 创建任务后长轮询返回任务
curl -s -w "%{http_code}" -o /dev/null \
  "http://localhost:17421/workers/{worker_id}/tasks/poll?timeout=5s" \
  -H "Authorization: Bearer $TOKEN"
# 期望输出: 200（有任务）

# Test 5: 上报结果
curl -s -X POST http://localhost:17421/tasks/{task_id}/result \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status":"completed","steps":[],"artifacts":[]}'
# 期望返回 200

# Test 6: token 撤销
curl -s -X POST http://localhost:17421/workers/{worker_id}/revoke
# 然后再次长轮询
curl -s -w "%{http_code}" -o /dev/null \
  "http://localhost:17421/workers/{worker_id}/tasks/poll" \
  -H "Authorization: Bearer $TOKEN"
# 期望输出: 401
```
