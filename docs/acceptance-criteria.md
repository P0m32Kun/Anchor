# Sprint 0.5 主流程验收用例

> 基于 `docs/v0.3-plan.md` 阶段目标与 Sprint 0 Scout 代码现状编写。
> 所有用例对齐当前代码实际能力，不做超出当前实现范围的假设。  
> ⚠️ 此文档已归档，路由结构已发生重大变更，仅供参考

---

## AC-001 创建项目

**前置条件**：
- 桌面端应用已启动，后端 API 可达。
- 当前位于 `/projects` 页面或可通过导航栏进入。

**操作步骤**：
1. 点击导航栏「项目管理」进入项目列表页。
2. 在创建表单中填写「项目名称」（必填）。
3. （可选）填写组织、目的、时间窗口、速率限制。
4. 点击「创建项目」按钮。

**预期结果**：
- 新创建的项目出现在列表顶部。
- 表单字段清空。
- 列表中项目卡片显示名称、组织、状态标签（进行中/未开始/已过期）。

**通过标准**：
- ✅ Pass：项目成功创建并实时出现在列表中，无页面刷新。
- ❌ Fail：点击后无响应、页面白屏、项目未出现或出现重复项目。

## AC-002 导入目标

**前置条件**：
- 已创建至少一个项目。
- 准备好一个 `.txt` 或 `.csv` 文件，每行一个目标（域名/IP）。

**操作步骤**：
1. 在 Dashboard 或项目列表中点击一个项目，将其设为当前项目（store 中的 `currentProject`）。
2. 进入「目标管理」(`/targets`) 页面。
3. 在文件导入区域点击上传或拖拽文件到虚线框内。
4. 等待导入完成。

**预期结果**：
- 导入区域显示「正在导入...」状态。
- 完成后显示导入结果统计（成功导入数、重复数、拒绝数、错误数）。
- 目标列表自动刷新，展示新增的目标及其类型标签。

**通过标准**：
- ✅ Pass：文件成功上传，统计数字正确，目标出现在列表中。
- ❌ Fail：上传无反应、报错无提示、目标未出现在列表、格式错误的文件导致页面崩溃。

## AC-003 Scope 确认

**前置条件**：
- 已进入某项目的目标管理页面 (`/targets`)，且该项目已有目标。

**操作步骤**：
1. 在 Scope 规则区域选择「包含」或「排除」。
2. 输入规则值（如域名），点击「添加规则」。
3. 点击「运行 Dry Run」按钮。

**预期结果**：
- Scope 规则添加后给出「规则已添加」提示。
- Dry Run 结果展示每个目标的决策（allowed/denied）及原因。
- 若目标超出时间窗口或被规则拒绝，决策显示为 denied 并附带原因。

**通过标准**：
- ✅ Pass：规则成功添加，Dry Run 结果列表渲染正常，决策与原因对应正确。
- ❌ Fail：规则添加无反馈、Dry Run 报错白屏、结果与预期不符。

## AC-004 启动扫描

**前置条件**：
- 当前项目已设置，且该项目下已有目标（至少一个域名目标用于子域名扫描）。
- 已进入「扫描任务」(`/runs`) 页面。

**操作步骤**：
1. 点击「新建扫描」按钮。
2. 选择扫描模板并输入扫描名称。
3. 点击「启动扫描」。

**预期结果**：
- 弹窗关闭，新建扫描出现在任务列表顶部，状态为「待启动」或「运行中」。
- 页面轮询或刷新后任务状态更新。

**通过标准**：
- ✅ Pass：扫描任务成功创建，列表中可见，状态标签颜色正确（黄/蓝/绿/红）。
- ❌ Fail：无响应、报错 alert、任务未出现在列表中。

## AC-005 查看扫描任务状态

**前置条件**：
- 已有至少一个扫描任务（任何状态）。

**操作步骤**：
1. 进入「扫描任务」(`/runs`) 页面。
2. 查看扫描列表中的状态列。
3. 点击某条扫描记录，展开其下属任务列表。

**预期结果**：
- 列表正确显示各扫描的状态（待启动 / 运行中 / 已完成 / 失败 / 已取消）。
- 展开后显示子任务（子域名发现、端口扫描、漏洞扫描等）及其状态。
- 状态标签使用对应颜色区分。

**通过标准**：
- ✅ Pass：状态显示正确，展开/收起交互正常，无数据时显示空状态提示。
- ❌ Fail：状态显示错误、点击展开无反应、子任务状态与父任务不一致。

## AC-006 查看资产清单

**前置条件**：
- 某项目已完成至少一轮扫描，产生了资产数据。

**操作步骤**：
1. 进入「资产清单」(`/assets`) 页面。
2. 切换 Tab：资产 / Web 端点 / 端口。
3. 点击某条资产查看详情（Web 端点状态码、端口服务等）。

**预期结果**：
- 资产列表按类型（domain / ip / url）显示徽章标签。
- Web 端点 Tab 显示 URL、状态码、Title、技术栈。
- 端口 Tab 显示关联端口与服务信息。
- 状态码徽章按区间着色（2xx 绿 / 3xx 黄 / 4xx/5xx 红）。

**通过标准**：
- ✅ Pass：三个 Tab 数据渲染正确，徽章颜色与状态码区间匹配，滚动加载正常。
- ❌ Fail：Tab 切换白屏、数据为空但无空状态提示、徽章颜色错误。

## AC-007 查看漏洞发现

**前置条件**：
- 某项目已完成漏洞扫描，产生了 findings 数据。

**操作步骤**：
1. 进入「漏洞发现」(`/findings`) 页面。
2. 在搜索框输入关键词进行过滤。
3. 点击某条 finding 查看详情。

**预期结果**：
- 列表显示漏洞名称、严重等级徽章（Critical 红 / High 橙 / Medium 黄 / Low 蓝 / Info 灰）。
- 过滤后列表实时响应，无全页刷新。
- 详情弹窗显示漏洞描述、证据列表。

**通过标准**：
- ✅ Pass：过滤功能正常，严重等级颜色正确，详情弹窗可正常打开和关闭。
- ❌ Fail：过滤无反应、弹窗无法关闭、详情数据缺失或报错。

## AC-008 导出报告

**前置条件**：
- 某项目下至少有一条 status 为 `confirmed` 或 `accepted_risk` 的 finding。

**操作步骤**：
1. 进入「报告导出」(`/reports`) 页面。
2. 等待报告预览加载完成。
3. 点击「导出 Markdown」或「导出 JSON」按钮。

**预期结果**：
- 页面加载后显示可导出的 findings 列表。
- 点击导出按钮后触发浏览器文件下载。
- 预览文本区域展示 Markdown 格式的报告内容。

**通过标准**：
- ✅ Pass：报告列表加载成功，导出按钮触发下载，文件内容完整。
- ❌ Fail：导出无反应、下载文件为空、页面报错、非确认状态的 finding 被包含在报告中。

## AC-009 错误状态处理（网络断开）

**前置条件**：
- 应用处于正常运行状态，当前位于任意项目内页面。

**操作步骤**：
1. 断开后端服务或切断网络连接。
2. 在当前页面执行任意需要 API 的操作（如切换 Tab、刷新列表）。
3. 观察页面表现。

**预期结果**：
- 页面不白屏。
- 控制台输出错误日志（当前无全局错误拦截，错误通过 `console.error` 输出）。
- 部分页面会显示局部错误文本（如 ReportsPage 的 `error` 状态）。
- 用户仍可进行纯本地交互（如切换 Tab、关闭弹窗）。

**通过标准**：
- ✅ Pass：页面保持渲染，不白屏，不崩溃；部分页面显示错误文本提示。
- ❌ Fail：页面白屏、React 渲染崩溃、无限 Loading 无反馈。

> **现状说明**：当前无全局 Error Boundary 和统一网络错误拦截，错误处理依赖各页面自行捕获。因此本用例 pass 标准适当放宽至「不白屏 + 有可见错误反馈」。

## AC-010 项目切换

**前置条件**：
- 已创建至少两个项目。

**操作步骤**：
1. 在 Dashboard (`/`) 查看项目列表。
2. 点击项目 A 的卡片或链接，进入目标管理页 (`/targets`)。
3. 返回 Dashboard，点击项目 B。
4. 再次进入目标管理页。

**预期结果**：
- 项目 A 的目标数据与项目 B 的目标数据互不混淆。
- 当前项目通过 `currentProject`（Zustand store）或 `useParams` 的 `id` 传递给各页面。
- 导航栏保持可见，页面上下文随项目切换而更新。

**通过标准**：
- ✅ Pass：切换项目后，各页面加载对应项目的数据，无数据残留或交叉污染。
- ❌ Fail：切换后仍显示上一个项目的数据、页面报错 404/500、路由跳转失败。

> **现状说明**：当前路由为 flat 结构（`/targets`、`/assets` 等），项目上下文通过 Zustand `currentProject` 或 legacy 路由 `/projects/:id/*` 的 params 传递。尚未实现 v0.3 计划中的嵌套路由 `/projects/:id/targets`。因此本用例按 legacy 路由 + store 切换的行为描述。

## 附录：当前路由与页面映射

| 页面 | 路由 | 数据来源 |
|------|------|----------|
| Dashboard | `/` | 全局 API（项目列表、Worker 状态） |
| 项目管理 | `/projects` | `api.listProjects` |
| 目标管理 | `/targets` | `useParams().id` 或 `currentProject.id` → `api.listTargets` |
| 资产清单 | `/assets` | `useParams().id` 或 `currentProject.id` → `api.listAssets` |
| 扫描任务 | `/runs` | `currentProject.id` → `api.listRuns` |
| 漏洞发现 | `/findings` | `useParams().id` → `api.listFindings` |
| 报告导出 | `/reports` | `useParams().id` → `api.listFindings`（过滤 confirmed） |
| Worker 管理 | `/workers` | `api.listWorkers` |
| 应用配置 | `/settings` | 本地状态 |
