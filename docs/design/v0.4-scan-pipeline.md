# v0.4 设计方案：智能扫描流程与多目标类型支持

> 状态：**In Review**  
> 日期：2026-05-02  
> 作者：Anchor Team
> 默认规则：这是候选设计稿，不是当前仓库基线；当前真相见 `docs/current/architecture.md`

---

## 1. 目标与背景

### 1.1 当前痛点

| 问题               | 现状                                    |
| ------------------ | --------------------------------------- |
| 目标类型有限       | 仅支持 domain/url/ip/cidr，不支持企业名 |
| 无互联网搜索集成   | 无法从公开数据源主动发现企业关联资产    |
| 无 CDN 过滤        | 直接扫描 CDN IP，浪费资源且结果不可靠   |
| 服务指纹依赖端口号 | 非 Web 服务仅靠端口映射，误判率高       |
| 扫描流程线性       | 不同目标类型走同一流程，无差异化处理    |

### 1.2 v0.4 目标

1. **多目标类型导入**：域名、IP、IP 段、企业名，自动分类并配置不同入口点
2. **FOFA 集成**：企业名 → FOFA 发现关联域名/IP（FOFA 仅用于企业名场景）
3. **完整扫描管线**：子域名发现 → DNS 解析 → CDN 过滤 → 端口扫描 → 服务指纹 → 漏洞探测
4. **智能服务指纹**：Web + 非 Web 服务识别，不依赖端口号
5. **指纹驱动漏洞探测**：服务指纹精确映射 Nuclei tags

## 2. 整体架构

### 2.1 扫描管线总览

```
                         ┌─────────────────────────────────────┐
                         │         Target Import & Classify    │
                         │  domain / ip / cidr / company       │
                         └──────────────┬──────────────────────┘
                                        │
          ┌─────────────┬───────────────┼───────────────┬─────────────┐
          ▼             ▼               ▼               ▼             ▼
   ┌────────────┐ ┌──────────┐  ┌────────────┐  ┌──────────┐ ┌──────────┐
   │  Company   │ │  Domain  │  │    IP      │  │   CIDR   │ │   URL    │
   │            │ │          │  │            │  │          │ │          │
   │ FOFA 搜索  │ │ Subfinder│  │ CDNcheck   │  │ Naabu    │ │ httpx    │
   │ ↓          │ │ ↓        │  │ ↓          │  │ ↓        │ │ ↓        │
   │ 展开域名/IP│ │ DNS 解析 │  │ Naabu 端口 │  │ nerva    │ │ Nuclei   │
   │ ↓          │ │ ↓        │  │ ↓          │  │ ↓        │ └──────────┘
   │ 进入对应   │ │ CDNcheck │  │ nerva      │  │ Nuclei   │
   │ Domain/IP  │ │ ↓        │  │ ↓          │  └──────────┘
   │ 流程       │ │ Naabu    │  │ Web → httpx│
   └────────────┘ │ ↓        │  │ 非Web →    │
                  │ nerva    │  Nuclei tags  │
                  │ ↓        │  └──────────┘
                  │ Web→httpx│
                  │ 非Web→   │
                  │ Nuclei   │
                  └──────────┘
```

### 2.2 核心设计原则

#### 原则 1：FOFA 仅用于企业名

| 目标类型 | 入口工具         | FOFA?       |
| -------- | ---------------- | ----------- |
| domain   | Subfinder        | ❌          |
| ip       | cdncheck → Naabu | ❌          |
| cidr     | Naabu            | ❌          |
| company  | FOFA             | ✅ 唯一入口 |
| url      | httpx            | ❌          |

#### 原则 2：端口扫描优先，服务指纹驱动后续流程

> **Naabu 发现开放端口 → nerva 识别服务类型 → 根据服务类型分流**

```
Naabu (开放端口) → nerva (服务指纹)
                         ↙          ↘
                    Web服务        非Web服务
                       ↓              ↓
                    httpx        直接映射Nuclei tags
                    (技术栈)         ↓
                       ↓          Nuclei 扫描
                    Nuclei 扫描
```

**关键**：httpx **不在** Naabu 之前，而是在 nerva 识别出 Web 服务**之后**才执行。

#### 原则 3：CDN 过滤在端口扫描之前

域名流程中，CDN 过滤在 DNS 解析之后、端口扫描之前执行，避免对 CDN IP 做无意义的端口扫描。

### 2.3 阶段定义（Pipeline Stages）

| 阶段               | ID            | 输入                        | 输出                     | 工具               |
| ------------------ | ------------- | --------------------------- | ------------------------ | ------------------ |
| **S0: 目标分类**   | `classify`    | 原始导入文本                | 分类后的 Target 列表     | 自研               |
| **S1: FOFA 搜索**  | `search`      | company                     | 域名/IP 列表             | FOFA API           |
| **S2: 子域名发现** | `subdomain`   | domain                      | 子域名列表               | Subfinder          |
| **S3: DNS 解析**   | `resolve`     | 域名列表                    | IP 列表 (含 CNAME)       | 自研 DNS resolver  |
| **S4: CDN 过滤**   | `cdn_filter`  | IP 列表                     | 非 CDN IP 列表           | cdncheck           |
| **S5: 端口扫描**   | `portscan`    | IP 列表                     | 开放端口列表             | Naabu              |
| **S6: 服务指纹**   | `fingerprint` | IP:Port 列表                | Service 列表 (Web/非Web) | nerva              |
| **S7: Web 探活**   | `httpx`       | Web 服务 IP:Port            | WebEndpoint (含技术栈)   | httpx              |
| **S8: 漏洞探测**   | `vuln`        | WebEndpoint + 非Web Service | Finding                  | Nuclei (tags 驱动) |

## 3. 实现状态

> 注：以下内容描述当前实现方向与目标状态，不代表已经完成最终验收。

### 3.1 已实现的包

| 包              | 路径                            | 职责                                             |
| --------------- | ------------------------------- | ------------------------------------------------ |
| Pipeline 编排器 | `internal/workflow/pipeline.go` | 目标分流 → 阶段调度 → 结果传递                   |
| FOFA 客户端     | `internal/search/fofa.go`       | FOFA API 封装，company/org/cert/title 多查询组合 |
| DNS 解析器      | `internal/resolve/resolver.go`  | 多 DNS 服务器并发解析，可配置并发/超时           |
| CDN 检测器      | `internal/cdn/detector.go`      | cdncheck CLI 集成，批量检测                      |
| 服务指纹        | `internal/fingerprint/nerva.go` | nerva CLI 集成，Web/非Web 分类                   |
| 工具解析器      | `internal/parsers/`             | subfinder/naabu/httpx JSONL 输出解析             |
| Tag 映射        | `internal/nuclei/tagmapper.go`  | 30+ 服务名 → Nuclei tags                         |
| 端口命令构建    | `internal/worker/commands.go`   | Naabu 命令构建（含 port range preset 支持）      |

### 3.2 前端实现

| 页面/组件 | 路径 | 职责 |
| --- | --- | --- |
| ScanConfigPage | `frontend/src/pages/ScanConfigPage.tsx` | 项目内扫描配置（端口预设、阶段开关、并发/超时） |
| PipelineConfig API | `frontend/src/lib/api.ts` | `getPipelineConfig` / `updatePipelineConfig` |
| PORT_RANGE_PRESETS | `frontend/src/lib/api.ts` | 5 种端口预设定义与描述 |

### 3.2 数据模型变更

新增模型：

- `TargetTypeCompany` — 企业名目标类型
- `PipelineConfig` — Pipeline 各阶段开关与并发配置
- `DNSRecord` — DNS 解析结果
- `CDNResult` — CDN 检测结果
- `ServiceFingerprint` — nerva 服务指纹结果

修改模型：

- `Project` — 删除 `StartTime`/`EndTime`，新增 `FofaEmail`/`FofaAPIKey`/`PipelineConfig`

### 3.3 端口范围预设

Naabu 端口扫描支持五种预设（项目级可配置）：

| 预设 | 值 | 说明 |
|------|-----|------|
| Top 100 | `top100` | Naabu 默认快扫 |
| Top 1000 | `top1000` | 更广覆盖 |
| **高危端口** | `high-risk` | 115 个攻击面端口（Redis 6379、ES 9200、MongoDB 27017、K8s 10250、Ollama 11434 等） |
| 全端口 | `full` | 1-65535 |
| 自定义 | `custom` | 手动输入端口列表 |

高危端口列表定义在 `internal/worker/commands.go:HighRiskPorts`。

### 3.4 数据库迁移

详见 `docs/design/v0.4-migrations.md`。关键变更：

- v4：新增 `company` target 类型、FOFA 字段、dns_records / cdn_results / service_fingerprints 表
- v5：删除 `projects.start_time` / `projects.end_time`

### 3.4 API 端点

| 方法 | 路径                             | 说明               |
| ---- | -------------------------------- | ------------------ |
| POST | `/projects/{id}/pipeline/run`    | 启动 Pipeline 扫描 |
| GET  | `/projects/{id}/pipeline/config` | 获取 Pipeline 配置 |
| POST | `/projects/{id}/pipeline/config` | 更新 Pipeline 配置 |
