# 内网扫描与容器化架构设计

## 1. 背景与问题

### 1.1 当前扫描流程的限制

当前资产发现工作流 `AssetDiscoveryWorkflow` 的链路为：

```
域名目标 → Subfinder → httpx → Naabu
```

这个设计有两个根本性问题：

1. **入口类型单一**：只处理 `TargetTypeDomain`，`ip` / `cidr` / `url` 类型的目标被直接忽略
2. **工具链强绑定域名**：Subfinder 是 DNS 子域枚举工具，在内网无 DNS 环境下必然返回空，导致后续 httpx/Naabu 链路中断

### 1.2 靶场验证暴露的问题

靶场环境（172.20.0.0/24）中有 5 个真实服务（nginx、tomcat、grafana、redis、mysql）。将 `172.20.0.0/24` 作为 CIDR target 导入后：

- Subfinder 链路不执行（无域名目标）
- IP/CIDR 目标被 `domainTargets` 过滤直接丢弃
- 没有任何端口扫描或漏洞验证被执行

### 1.3 容器化不一致

当前存在三种运行方式：
- 本地开发：`go run .` 直接启动 Server + 内嵌 Worker
- 远程部署：`Dockerfile.server` / `Dockerfile.worker` 单独构建
- 靶场测试：`docker-rangefield/docker-compose.yml` 独立网络

三种方式互不兼容，内网扫描测试无法真实模拟。

---

## 2. 设计目标

1. **支持 IP/CIDR 作为扫描入口**：内网环境下可以直接输入网段或 IP 列表开始扫描
2. **支持自定义端口范围**：用户可以手动限制扫描端口，工具根据 profile 自动选择合适的默认范围
3. **统一容器化**：Server、Worker、靶场全部容器化，通过共享 Docker 网络模拟真实内网环境
4. **桌面端连接不变**：桌面端仍在宿主机运行，通过 `localhost:17421` 连接容器化 Server
5. **运行方式统一**：本地开发和远程 VPS 部署使用同一套容器配置

### 3.6 目标输入格式扩展

当前导入解析器 `ParseTXT` / `ParseCSV` 存在的问题：
1. **无类型前缀时默认 domain**：`127.0.0.1` 会被当成域名
2. **不处理逗号分隔**：一行中多个 IP 用逗号分隔时被视为一个值
3. **不处理连字符范围**：`192.168.0.1-10` 无法展开为多个 IP

#### 3.6.1 自动类型推断

当用户未提供 `ip:` / `cidr:` / `domain:` / `url:` 前缀时，根据值的内容自动推断类型：

| 输入值 | 推断类型 | 原因 |
|--------|---------|------|
| `192.168.1.1` | `ip` | 匹配 IPv4 地址格式 |
| `10.0.0.0/8` | `cidr` | 匹配 CIDR 格式（`x.x.x.x/y`） |
| `example.com` | `domain` | 不是 IP 也不是 CIDR，默认域名 |
| `https://example.com` | `url` | 以 `http://` 或 `https://` 开头 |
| `192.168.0.1-10` | `ip`（展开后） | 匹配连字符范围格式 |

```go
// detectType 根据值内容自动推断目标类型
func detectType(val string) models.TargetType {
    // URL 前缀
    if strings.HasPrefix(val, "http://") || strings.HasPrefix(val, "https://") {
        return models.TargetTypeURL
    }
    // CIDR 格式
    if strings.Contains(val, "/") {
        if _, _, err := net.ParseCIDR(val); err == nil {
            return models.TargetTypeCIDR
        }
    }
    // IPv4 地址（包含连字符范围，如 192.168.0.1-10）
    if isIPv4OrRange(val) {
        return models.TargetTypeIP
    }
    // 默认域名
    return models.TargetTypeDomain
}

// isIPv4OrRange 判断是否 IPv4 地址或连字符范围（如 192.168.0.1 或 192.168.0.1-10）
func isIPv4OrRange(s string) bool {
    // 支持 "192.168.0.1" 和 "192.168.0.1-10"
    parts := strings.Split(s, ".")
    if len(parts) != 4 {
        return false
    }
    // 最后一段可能是 "1" 或 "1-10"
    lastPart := parts[3]
    if strings.Contains(lastPart, "-") {
        rangeParts := strings.SplitN(lastPart, "-", 2)
        lastPart = rangeParts[0]
    }
    parts[3] = lastPart
    ip := strings.Join(parts, ".")
    return net.ParseIP(ip) != nil && strings.Contains(ip, ".")
}
```

#### 3.6.2 逗号分隔展开

一行中包含逗号时，视为多个独立目标，每个分别解析：

```go
// expandCommas 将逗号分隔的字符串展开为多个值
// 输入: "192.168.0.1,192.168.0.2,192.168.0.3"
// 输出: ["192.168.0.1", "192.168.0.2", "192.168.0.3"]
func expandCommas(line string) []string {
    parts := strings.Split(line, ",")
    var results []string
    for _, p := range parts {
        p = strings.TrimSpace(p)
        if p != "" {
            results = append(results, p)
        }
    }
    return results
}
```

**示例**：

```
# TXT 导入文件
192.168.0.1,192.168.0.2,192.168.0.3   → 3 个 ip 目标
ip:10.0.0.1,10.0.0.2                  → 2 个 ip 目标（前缀共享）
domain:example.com,sub.example.com    → 2 个 domain 目标
```

#### 3.6.3 连字符范围展开

支持 IPv4 地址的最后一段使用连字符表示范围：

```go
// expandIPRange 将 "192.168.0.1-10" 展开为 10 个 IP
func expandIPRange(s string) ([]string, error) {
    // 检查是否包含连字符
    if !strings.Contains(s, "-") {
        return []string{s}, nil  // 不是范围，原样返回
    }

    parts := strings.Split(s, ".")
    if len(parts) != 4 {
        return nil, fmt.Errorf("invalid IPv4 range: %s", s)
    }

    lastPart := parts[3]
    rangeParts := strings.SplitN(lastPart, "-", 2)
    if len(rangeParts) != 2 {
        return []string{s}, nil
    }

    start, err := strconv.Atoi(rangeParts[0])
    if err != nil {
        return nil, fmt.Errorf("invalid range start: %s", rangeParts[0])
    }
    end, err := strconv.Atoi(rangeParts[1])
    if err != nil {
        return nil, fmt.Errorf("invalid range end: %s", rangeParts[1])
    }

    if start > end || start < 0 || end > 255 {
        return nil, fmt.Errorf("invalid range: %d-%d", start, end)
    }

    prefix := strings.Join(parts[:3], ".") + "."
    var results []string
    for i := start; i <= end; i++ {
        results = append(results, fmt.Sprintf("%s%d", prefix, i))
    }
    return results, nil
}
```

**支持的范围格式**：

| 输入 | 展开结果 | 数量 |
|------|---------|------|
| `192.168.0.1-3` | `192.168.0.1`, `192.168.0.2`, `192.168.0.3` | 3 |
| `10.0.1.100-103` | `10.0.1.100` 到 `10.0.1.103` | 4 |
| `172.16.0.1-1` | `172.16.0.1` | 1（边界情况） |

**不支持的情况**（超出范围时返回错误，由调用方处理）：

| 输入 | 行为 |
|------|------|
| `192.168.0.300-305` | 超出 0-255，返回错误 |
| `192.168.0.10-5` | start > end，返回错误 |
| `192.168.0.1-192.168.0.10` | 不支持跨段范围，视为一个域名目标 |

#### 3.6.4 完整解析流程

```go
// parseLine 重构后的完整逻辑
func parseLine(line string) []ImportTarget {
    line = strings.TrimSpace(line)
    if line == "" {
        return nil
    }

    // 1. 提取类型前缀（如果存在）
    var explicitType models.TargetType
    var val string
    if idx := strings.Index(line, ":"); idx > 0 {
        typ := strings.ToLower(strings.TrimSpace(line[:idx]))
        v := strings.TrimSpace(line[idx+1:])
        switch typ {
        case "domain":
            explicitType = models.TargetTypeDomain
            val = v
        case "url":
            explicitType = models.TargetTypeURL
            val = v
        case "ip":
            explicitType = models.TargetTypeIP
            val = v
        case "cidr":
            explicitType = models.TargetTypeCIDR
            val = v
        default:
            // 未识别的类型前缀，视为无前缀
            explicitType = ""
            val = line
        }
    } else {
        val = line
    }

    // 2. 逗号分隔展开
    parts := expandCommas(val)

    // 3. 逐段解析
    var targets []ImportTarget
    for _, part := range parts {
        part = strings.TrimSpace(part)
        if part == "" {
            continue
        }

        // 确定类型
        typ := explicitType
        if typ == "" {
            typ = detectType(part)
        }

        // 4. IP 范围展开（仅对 ip 类型）
        if typ == models.TargetTypeIP {
            ips, err := expandIPRange(part)
            if err == nil {
                for _, ip := range ips {
                    targets = append(targets, ImportTarget{Type: models.TargetTypeIP, Value: ip})
                }
                continue
            }
            // 范围展开失败，回退为单个值
        }

        targets = append(targets, ImportTarget{Type: typ, Value: part})
    }

    return targets
}
```

为防止用户误操作导致导入过多目标，设置以下限制：

```go

    maxImportBatch = 10000   // 单次导入最多 10000 个目标
)
```

| 场景 | 行为 |
|------|------|
| `192.168.0.1-300` | 超出 255 范围，返回错误，该条目标被拒绝 |
| `10.0.0.1-256` | 超出 maxIPRangeSize(256)，返回错误 |

## 4. Scope 引擎扩展

### 4.1 IP/CIDR 范围校验（已有基础）

当前 `scope.Engine.matchIP()` 已支持：
- `TargetTypeIP`：精确匹配
- `TargetTypeCIDR`：网段包含判断

**新增需求**：CIDR target 导入后，扫描产生的具体 IP 需要经过 Scope 校验。

### 4.2 CIDR 展开策略

```go
// ExpandCIDR 将 CIDR 展开为 IP 列表（用于全端口扫描前的确认）
func ExpandCIDR(cidr string) ([]string, error) {
    ip, network, err := net.ParseCIDR(cidr)
    if err != nil {
        return nil, err
    }
    
    var ips []string
    for ip := ip.Mask(network.Mask); network.Contains(ip); incrementIP(ip) {
        // 跳过网络地址和广播地址（对于 /24 及以下）
        if isNetworkOrBroadcast(ip, network) {
            continue
        }
        ips = append(ips, ip.String())
    }
    return ips, nil
}
```

### 4.3 Scope 校验策略

对于 IP/CIDR 类型的扫描目标，Scope 校验分两个层次：

1. **父级目标校验**：导入 CIDR `172.20.0.0/24` 时，检查 CIDR 是否在授权范围内
2. **子级结果校验**：扫描产生的具体 IP（如 `172.20.0.10`）需要再次经过 Scope 校验

```go
// 伪代码：CIDR 扫描时的 Scope 校验
func (e *Engine) CheckIP(ctx context.Context, projectID, ip string) (*ScopeDecision, error) {
    target := &models.Target{
        Type:  models.TargetTypeIP,
        Value: ip,
    }
    return e.Check(ctx, projectID, target)
}
```

### 5.3 工作流核心逻辑变更

```go
func (w *AssetDiscoveryWorkflow) Run(ctx context.Context, projectID string) (*DiscoveryResult, error) {
    targets, _ := w.queries.ListTargetsByProject(projectID)
    
    // 按类型分组
    var domainTargets, ipTargets, cidrTargets []*models.Target
    for _, t := range targets {
        switch t.Type {
        case models.TargetTypeDomain:
            domainTargets = append(domainTargets, t)
        case models.TargetTypeIP:
            ipTargets = append(ipTargets, t)
        case models.TargetTypeCIDR:
            cidrTargets = append(cidrTargets, t)
        }
    }
    
    // 1. 域名链路（已有，保持不变）
    if len(domainTargets) > 0 {
        w.runDomainChain(ctx, projectID, domainTargets)
    }
    
    // 2. IP 链路（新增）
    if len(ipTargets) > 0 {
        w.runIPChain(ctx, projectID, ipTargets)
    }
    
    // 3. CIDR 链路（新增）
    if len(cidrTargets) > 0 {
        w.runCIDRChain(ctx, projectID, cidrTargets)
    }
    
    // 4. 统一后续：Web 初筛、漏洞扫描
    w.runPostDiscovery(ctx, projectID)
    
    return result, nil
}
```

### 5.3.1 Scope 校验覆盖策略

三条链路的 Scope 校验策略不同：

| 链路 | 校验点 | 校验对象 | 说明 |
|------|--------|---------|------|
| **域名** | Target 创建/导入时 + 工作流运行时 | 域名本身 | 已有逻辑，不变 |
| **IP** | Target 创建/导入时 + 工作流运行时 | 单个 IP | **新增**：工作流启动前校验，扫描结果就是这个 IP 本身，无需二次校验 |
| **CIDR** | Target 创建/导入时 + 工作流运行时 + 扫描结果逐个 | CIDR 网段 + 展开后的每个 IP | **新增**：CIDR 作为父级目标先校验，展开后的每个具体 IP 再逐个校验 |

### 5.4 IP 链路详细流程

```go
func (w *AssetDiscoveryWorkflow) runIPChain(ctx context.Context, projectID string, ipTargets []*models.Target) error {
    for _, it := range ipTargets {
        // 1. Scope 校验 IP 目标本身（工作流运行时再次校验）
        decision, _ := w.scope.Check(ctx, projectID, it)
        if decision.Decision == models.ScopeDeny {
            continue
        }
        
        // 2. 写入 host 文件（单 IP）
        hostFile, _ := writeHostsFile(w.dataDir, projectID, []string{it.Value})
        
        // 3. Naabu 端口扫描（使用自定义端口范围）
        portRange := w.getPortRange(projectID)
        naabuTask, _ := w.createAndRunTask(ctx, projectID, it.ID, "naabu",
            worker.BuildNaabuCommand(hostFile, portRange))
        
        // 4. 解析结果
        naabuResults, _ := w.parseNaabuOutput(naabuTask.ID)
        
        // 5. 资产归并（IP 本身已通过 Scope 校验，结果无需二次校验）
        for _, nr := range naabuResults {
            a, _, _ := w.merger.MergeOrCreateAsset(projectID, "ip", nr.IP, "naabu")
            w.merger.MergeOrCreatePort(a.ID, nr.Port, "naabu")
        }
        
        // 6. 对存活 IP 运行 httpx
        if len(naabuResults) > 0 {
            httpxFile, _ := writeHostsFile(w.dataDir, projectID, []string{it.Value})
            httpxTask, _ := w.createAndRunTask(ctx, projectID, it.ID, "httpx",
                worker.BuildHttpxCommand(httpxFile))
            // ... 解析 WebEndpoint，归并资产
        }
    }
    return nil
}
```

**注意**：IP 链路中，目标本身就是具体的单个 IP，导入时已做过 Scope 校验，工作流运行时再校验一次。扫描发现的 port 属于同一个 IP，不需要对 port 本身做 Scope 校验（Scope 规则目前只针对 IP/CIDR/域名级别，不针对端口级别）。

### 5.5 CIDR 链路详细流程

```go
func (w *AssetDiscoveryWorkflow) runCIDRChain(ctx context.Context, projectID string, cidrTargets []*models.Target) error {
    for _, ct := range cidrTargets {
        // 1. Scope 校验 CIDR 本身（父级目标）
        decision, _ := w.scope.Check(ctx, projectID, ct)
        if decision.Decision == models.ScopeDeny {
            continue
        }
        
        // 2. 展开 CIDR 为 IP 列表
        ips, err := ExpandCIDR(ct.Value)
        if err != nil {
            continue
        }
        
        // 3. 写入 host 文件
        hostFile, _ := writeHostsFile(w.dataDir, projectID, ips)
        
        // 4. Naabu 端口扫描（使用自定义端口范围）
        portRange := w.getPortRange(projectID)
        naabuTask, _ := w.createAndRunTask(ctx, projectID, ct.ID, "naabu", 
            worker.BuildNaabuCommand(hostFile, portRange))
        
        // 5. 解析结果
        naabuResults, _ := w.parseNaabuOutput(naabuTask.ID)
        
        for _, nr := range naabuResults {
            // 6. 【关键】每个扫描发现的 IP 做二次 Scope 校验
            //    原因：CIDR 是粗粒度授权，展开后的具体 IP 可能有排除规则
            tmpTarget := &models.Target{Type: models.TargetTypeIP, Value: nr.IP}
            decision, _ := w.scope.Check(ctx, projectID, tmpTarget)
            if decision.Decision == models.ScopeDeny {
                continue  // 这个 IP 被排除，跳过
            }
            
            // 7. 资产归并
            a, _, _ := w.merger.MergeOrCreateAsset(projectID, "ip", nr.IP, "naabu")
            w.merger.MergeOrCreatePort(a.ID, nr.Port, "naabu")
        }
        
        // 8. 对通过 Scope 校验的存活 IP 运行 httpx
        aliveIPs := w.extractAliveIPs(naabuResults)
        // 注意：aliveIPs 已经从 naabuResults 过滤而来，只包含通过 Scope 校验的 IP
        if len(aliveIPs) > 0 {
            httpxFile, _ := writeHostsFile(w.dataDir, projectID, aliveIPs)
            httpxTask, _ := w.createAndRunTask(ctx, projectID, ct.ID, "httpx",
                worker.BuildHttpxCommand(httpxFile))
            // ... 解析 WebEndpoint，归并资产
        }
    }
    return nil
}
```

func BuildNaabuCommand(hostFile string, portRange string) []string {

## 11. 注意事项

### 11.1 启动顺序

靶场依赖 `anchor-net` 网络，必须先启动根目录 compose：

```bash
make up        # 创建网络
make range-up  # 靶场加入网络
# 或 make up-all 自动处理
```

### 11.2 权限

Worker 需要 `privileged: true` 和 `cap_add`：
- `SYS_ADMIN`：Chromium/Rod 截图
- `NET_RAW`：naabu 原始套接字扫描
- `NET_ADMIN`：网络管理

macOS Docker Desktop 中某些 capability 可能受限，基础扫描能力应该可用。

### 11.3 网络隔离

所有容器共享 `172.20.0.0/24` 子网：
- 靶场服务固定 IP：172.20.0.10-14
- Server/Worker 动态分配 IP（如 172.20.0.2、172.20.0.3）
- Worker 扫描 `172.20.0.0/24` 会扫到自己，但不影响功能

### 11.4 Nuclei Templates

Worker 镜像在 build 阶段下载模板。运行时更新：

```bash
docker-compose exec worker nuclei -update-templates
```

### 11.5 数据迁移

如果已有本地 SQLite 数据需要迁移到容器：

```bash
# 1. 备份本地数据
cp ~/.anchor/anchor.db ./anchor.db.bak

# 2. 启动容器后复制数据
docker cp ./anchor.db.bak anchor-server:/data/anchor.db
```
