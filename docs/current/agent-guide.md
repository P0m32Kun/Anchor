# Coding Agent Guide

This is the default operating guide for agents working in this repository.

## Reading Order

1. `README.md` for product shape and local commands.
2. `docs/README.md` for document navigation and lifecycle rules.
3. `docs/current/plan.md` for the only repository-wide plan.
4. `docs/current/architecture.md` for the architecture baseline.
5. `docs/current/deployment.md` for customer Docker deployment; `docs/current/e2e-testing.md` for developer E2E stacks (do not mix compose files).
6. `docs/current/design/README.md` only when a task explicitly targets an in-review proposal.

Do not treat archived plans, review notes, redirect files, or proposal documents as current truth unless the user explicitly asks for that context.

## Change Protocol

For code changes:

1. Check CodeGraph freshness with `codegraph status`.
2. Before editing a function, class, or method, run:

   ```bash
   codegraph impact <symbol>
   ```

3. If CodeGraph reports HIGH or CRITICAL risk, stop and warn the user before editing.
4. Prefer small, isolated changes with matching tests.

For documentation-only changes, keep edits scoped to the current document map and avoid changing product claims without checking implementation or tests.

## Validation Gates

Compiling is not done. For product behavior, E2E is the acceptance gate.

Use the smallest meaningful validation first:

```bash
go test ./...
cd frontend && npm run typecheck
cd frontend && npm run test:e2e
```

If E2E cannot run because Docker, browsers, or external tools are missing, report the exact blocker and list the remaining verification.

## Where New Information Goes

| Information type | Put it here |
| --- | --- |
| Current repository-wide plan | `docs/current/plan.md` |
| Current architecture baseline | `docs/current/architecture.md` |
| In-review design proposal | `docs/design/` |
| Accepted design index entry | `docs/current/design/README.md` |
| Historical milestone material | `docs/archived/` |
| Engineering conventions and pitfalls | `docs/conventions/` · `docs/pitfalls/` |
| Short-lived task notes | Existing task issue or a scoped task doc, not a new top-level plan |

## Local Artifact Hygiene

- Do not commit generated binaries, databases, Playwright state, reports, or runtime workdirs.
- Playwright `storage-state.json` is generated by `frontend/e2e/global-setup.ts` and removed by `frontend/e2e/global-teardown.ts`.
- If a local artifact is required to run a test, generate it from checked-in code instead of documenting a manual pre-step.

## Agent-Friendly Defaults

- Prefer current docs over historical docs.
- Prefer CodeGraph search/context/impact over raw text search for unfamiliar code.
- Keep interfaces boring and explicit; optimize for the next agent being able to find the owner, invariant, and test.
- When changing behavior, update the relevant E2E test in the same change.
