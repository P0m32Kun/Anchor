## 前言

本手册面向**电力行业内网**渗透测试场景（工作站、Linux 服务器、专用设备、工控设备），为渗透测试团队提供标准化的漏洞检测方法和修复建议。

### 手册设计原则

1. **远程黑盒测试**：所有命令均在测试机上发起远程网络请求，**严禁通过合法凭据登录目标主机进行本地审计**；nuclei `code/` 协议模板仅在调用测试机本机工具完成远程漏洞验证时使用，不预置任何脚本到目标主机。
2. **一问题一节**：每个 `###` 条目只描述一个明确的安全问题，不混排多个测试方向。
3. **一命令一验证**：每个条目只给**一条核心验证命令**，操作者无需选择工具或参数。
4. **优先 Nuclei**：所有条目优先使用 Nuclei 模板验证，仅当 nuclei 模板库无覆盖时使用 nmap 替代（已在对应条目标注）。
5. **弱口令极小字典**：弱口令/默认凭据检测使用**自定义极小字典模板**（2 用户名 × 2 密码，最多 4 次尝试），确保不影响设备稳定运行。
6. **结果可审计**：所有命令均包含结果导出参数（`-o <文件名>.json -j`），测试过程可追溯、可复查。
7. **模板可验证**：本手册引用的所有 Nuclei 模板均经过现场验证，确保在 `~/nuclei-templates/` 目录下真实存在，类型包括 `network/` `http/` `javascript/` `ssl/` 远程协议模板及用于远程漏洞验证的 `code/` 协议模板。

### 风险等级定义

| 等级 | 标识 | 定义 |
|------|------|------|
| 高 | 🔴 | 可导致系统完全失控、敏感数据大规模泄露或内网横向移动 |
| 中 | 🟡 | 可导致局部权限提升、信息泄露或成为进一步攻击的跳板 |
| 低 | 🟢 | 主要造成信息泄露或攻击面扩大，单独利用难以直接造成实质危害 |

### 工具版本要求

| 工具 | 最低版本 | 备注 |
|------|----------|------|
| Nuclei | v3.0+ | 支持 `-j` JSON 输出和 JavaScript 模板 |
| Nmap | 7.92+ | 确保脚本数据库完整（仅 9.2 条目使用） |

### 通用参数约定

本手册中所有命令均遵循以下导出规范：

- **Nuclei**: `-o <文件名>_$(date +%Y%m%d).json -j`
- **Nmap**: `-oX <文件名>_$(date +%Y%m%d).xml`
- **MSF**：`spool <文件名>_$(date +%Y%m%d).txt`

### 安全红线（强制执行）

以下行为在本手册中**严格禁止**：

1. ❌ **禁止登录目标主机或在目标主机上执行任何本地命令**（仅允许测试机发起远程网络扫描）
2. ❌ **禁止利用漏洞获取 Shell 或执行任意命令**
3. ❌ **禁止修改目标系统的任何配置或数据**
4. ❌ **禁止对目标系统造成拒绝服务（DoS）**
5. ❌ **禁止密码喷洒超过手册规定的字典规模**（最大 4 次尝试）
6. ❌ **禁止在不授权的系统上执行测试**
7. ❌ **禁止将测试结果中的敏感信息记录到非加密存储**

### 变量说明

手册命令中的变量约定：

| 变量 | 含义 | 示例 |
|------|------|------|
| `<target>` | 目标主机 IP 或域名 | `192.168.1.100` |

每个条目的"结果说明"采用统一格式：

- `[模板ID] [严重级别] [协议] 目标:端口` → **存在漏洞，需立即修复**
- 无匹配结果 → **未检出（不代表绝对安全，仅说明当前检测条件未命中）**

# 第1章 网络基础设施层

---

### 1.1 SNMP 公共团体字符串

**风险等级**: 🔴 高

**漏洞描述**
> SNMP v1/v2c 使用默认或弱团体字符串（如 public/private），攻击者可读取设备配置、性能指标、路由表等敏感信息。

**受影响服务**
- 服务: SNMP
- 默认端口: UDP 161

**验证命令**
```bash
nuclei -t ~/nuclei-templates/javascript/udp/misconfiguration/snmpv1-community-detect-string.yaml -u <target> -o snmp_community_$(date +%Y%m%d).json -j
```

**结果说明**

- `[模板ID] [严重级别] [协议] 目标:端口` → 存在漏洞
- 无匹配 → 未检出（不代表绝对安全）

**修复建议**
1. 升级到 SNMP v3 并使用基于用户的安全模型（USM）进行认证和加密。
2. 若必须使用 SNMP v1/v2c，修改默认团体字符串为高强度随机字符串，并限制访问源 IP。

---

### 1.2 NTP 信息泄露与 monlist 放大

**风险等级**: 🟢 低

**漏洞描述**
> NTP 服务暴露于网络且启用了版本查询或 monlist 功能，攻击者可获取服务器系统信息（操作系统、处理器类型等），或利用 monlist 功能发起 NTP 放大拒绝服务攻击。

**受影响服务**
- 服务: NTP
- 默认端口: UDP 123

**验证命令**
```bash
# 检测 NTP 版本及变量枚举
nuclei -t ~/nuclei-templates/javascript/udp/misconfiguration/ntp-enum-variables-enabled.yaml -u <target>:123 -o ntp_enum_$(date +%Y%m%d).json -j
```

> 💡 本条目 nuclei 模板覆盖变量枚举检测；monlist 功能需配合 nmap NSE 脚本二次验证：
> ```bash
> nmap -sU -p 123 --script ntp-monlist <target>
> ```

**结果说明**
- `[模板ID] [严重级别] [协议] 目标:端口` → 存在 NTP 信息泄露
- 无匹配 → 未检出（不代表绝对安全）

**修复建议**
1. 在 `ntp.conf` 中禁用 `monlist` 和 `mrulist` 功能（`restrict default noquery`）。
2. 限制 NTP 查询来源 IP，仅允许受信任的 NTP 客户端。
3. 升级至 ntpd 4.2.7p26 及之后版本。

### 2.2 SSH 弱算法配置

**风险等级**: 🟡 中

**漏洞描述**
> SSH 服务支持弱加密算法（如 RC4、DES、MD5），可能导致通信内容被解密或中间人攻击。

**受影响服务**
- 服务: OpenSSH
- 默认端口: TCP 22

**验证命令**
```bash
nuclei -t ~/nuclei-templates/javascript/misconfiguration/ssh/ssh-weak-algo-supported.yaml -u <target>:22 -o ssh_weak_algo_$(date +%Y%m%d).json -j
```

**结果说明**
- `[模板ID] [严重级别] [协议] 目标:端口` → 存在漏洞
- 无匹配 → 未检出（不代表绝对安全）

**修复建议**
1. 在 `sshd_config` 中禁用弱算法，仅保留 AES-GCM、ChaCha20-Poly1305 等现代算法。
2. 定期使用 `ssh-audit` 等工具审计服务器 SSH 配置。

- 服务: OpenSSH
- 默认端口: TCP 22

**验证命令**
```bash
nuclei -t ~/nuclei-templates/RBKD-templates/ssh-mini-brute.yaml -u <target>:22 -o ssh_weak_$(date +%Y%m%d).json -j
```

**结果说明**
- `[模板ID] [严重级别] [协议] 目标:端口` → 存在漏洞
- 无匹配 → 未检出（不代表绝对安全）

**修复建议**
1. 禁用密码认证，强制使用基于密钥的认证方式。
2. 若必须使用密码认证，实施强密码策略并启用多因素认证（MFA）。

### 2.7 OpenSSH 用户枚举

**风险等级**: 🟡 中

**漏洞描述**
> OpenSSH 7.7 及之前版本中存在用户枚举漏洞（CVE-2018-15473）。程序对有效用户名和无效用户名的身份验证请求返回不同的响应时间或错误信息，攻击者可据此枚举系统中的有效用户账号。

**受影响服务**
- 服务: OpenSSH
- 默认端口: TCP 22

**验证命令**

> ⚠️ nuclei 暂无专门 CVE-2018-15473 检测模板，需使用 nmap 或 Metasploit 辅助模块验证。

```bash
# 方法一：nmap NSE 脚本（优先）
nmap -p 22 --script ssh2-enum-algos,ssh-hostkey <target>

# 方法二：Metasploit 辅助模块
msfconsole -q -x "use auxiliary/scanner/ssh/ssh_enumusers; set RHOSTS <target>; set RPORT 22; run; exit"
```

**结果说明**
- nmap 显示 SSH 版本 ≤ 7.7 → 版本可能受影响，建议进一步用 msf 验证
- msf 输出含 `[+]` 及用户名列表 → 存在用户枚举漏洞
- 无用户列表输出 → 未检出（不代表绝对安全）

**修复建议**
1. 升级 OpenSSH 至 7.8 或更高版本。
2. 在 OpenSSH 配置中启用 `UsePAM yes` 和 `Delay 3` 以模糊化响应时间。
3. 部署 fail2ban 等工具限制 SSH 枚举行为。

### 2.8 X11 未授权访问

**风险等级**: 🔴 高

**漏洞描述**
> X Window 系统（X11）监听 TCP 6000 端口（显示号 :0），若未配置访问控制（xhost +），攻击者可无需认证直接连接并获取屏幕截图、模拟键盘鼠标输入，实现远程桌面控制。

**受影响服务**
- 服务: X11 Server
- 默认端口: TCP 6000+（6000 对应 :0，6001 对应 :1，以此类推）

**验证命令**
```bash
nuclei -t ~/nuclei-templates/javascript/misconfiguration/x11/x11-unauth-access.yaml -u <target>:6000 -o x11_unauth_$(date +%Y%m%d).json -j
```

**结果说明**
- `[模板ID] [严重级别] [协议] 目标:端口` → 存在漏洞
- 无匹配 → 未检出（不代表绝对安全）

**修复建议**
1. 在 X11 启动参数中添加 `-nolisten tcp`，禁用 TCP 监听。
2. 使用 `xhost -` 清空访问列表，仅通过 `xhost +<IP>` 允许特定来源。
3. 通过 SSH X11 转发（`-X`）替代直接暴露 X11 端口。

### 2.9 MS12-020 RDP 远程代码执行

**风险等级**: 🔴 高

**漏洞描述**
> Windows 远程桌面协议（RDP）在处理特定报文时存在内存访问错误，攻击者可发送特制 RDP 报文触发任意代码执行，无需用户交互即可控制目标系统。

**受影响服务**
- 服务: Microsoft RDP
- 默认端口: TCP 3389

**验证命令**

> nuclei 暂无对应模板，使用 nmap NSE 脚本验证：

```bash
nmap -p 3389 --script rdp-vuln-ms12-020 <target> -oX rdp_ms12_020_$(date +%Y%m%d).xml
```

**结果说明**
- 脚本输出含 `VULNERABLE` → 存在 MS12-020 漏洞
- 输出含 `NOT VULNERABLE` → 已修复或未受影响
- 无 RDP 服务响应 → 未检出

**修复建议**
1. 安装微软安全更新 KB2671387。
2. 关闭不必要的 RDP 服务，或通过网络 ACL 限制 3389 端口访问源。

### 2.10 CVE-2019-0708 BlueKeep RDP 远程代码执行

**风险等级**: 🔴 高

**漏洞描述**
> Windows 远程桌面服务存在远程代码执行漏洞（BlueKeep），攻击者无需身份认证即可通过发送特制 RDP 请求触发漏洞。该漏洞具有蠕虫化传播潜力，影响 Windows XP/7/Server 2003/2008/2008 R2。

**受影响服务**
- 服务: Microsoft RDP / Terminal Services
- 默认端口: TCP 3389

**验证命令**

> ⚠️ nuclei 与 nmap 均暂无专门检测脚本，需使用 Metasploit 辅助模块：

```bash
msfconsole -q -x "use auxiliary/scanner/rdp/cve_2019_0708_bluekeep; set RHOSTS <target>; run; exit"
```

**结果说明**

- 输出含 `VULNERABLE` → 存在 BlueKeep 漏洞
- 输出含 `SAFE` 或 `UNKNOWN` → 未检出或无法确认

**修复建议**
1. 安装微软官方 KB4499175 等补丁（对应各操作系统版本）。
2. 禁用 RDP 服务或在网络层阻断 3389 端口。
3. 启用网络级认证（NLA），虽不能完全阻止但可提高利用门槛。

### 2.11 CVE-2026-24061 GNU Inetutils Telnetd 认证绕过

**风险等级**: 🔴 高

**漏洞描述**
